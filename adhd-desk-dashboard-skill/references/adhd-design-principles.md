# ADHD/TDAH-Friendly Design Principles

| Principle | Implementation |
|---|---|
| Reduce working-memory load | Use one objective per page and visible next action fields. |
| Externalize task state | Use paper cards, tables, checkboxes, and Linear issue views. |
| Make completion explicit | Every workflow has a Definition of Done. |
| Reduce visual noise | No decorative elements; only functional boxes and tables. |
| Support dyslexia constraints | Large sans-serif text, wide spacing, short labels, high contrast. |
| Support task initiation | Use small first actions: open inbox, choose one item, classify one source. |
| Support recovery after interruption | Include save location, status, and next action on every page. |

## Labels

| Label | Meaning |
|---|---|
| FACT | Verified or source-backed information. |
| HYPOTHESIS | Plausible but unvalidated interpretation. |
| DECISION | Choice made by the user or project. |
| ACTION | Executable next step. |
