# Workflow Protocol

| Phase | Question | Output |
|---|---|---|
| 1. Parse | What is the user really asking to operate? | Scope statement. |
| 2. Segment | Which workflows are distinct? | MECE workflow map. |
| 3. Constrain | What cognitive/accessibility limits matter? | Design constraints. |
| 4. Materialize | What must be visible on paper? | A4 dashboard pages. |
| 5. Digitize | What belongs in Linear/Drive/Calendar? | Backlog and save map. |
| 6. Validate | Can the user execute without rereading the whole brief? | DoD and print test. |

## Default Workflows

| Workflow | Time Box | Input | Output | DoD |
|---|---:|---|---|---|
| DataClean | 45 min/day | Raw notes/chats/files | Classified knowledge | Inbox reduced, facts saved, actions routed. |
| Content Creation | Morning | Topic/current insight | Reusable content batch | Published or scheduled; URLs saved. |
| Ops Admin | 90 min/day | Login/email/accounts/backlog | Admin state clean | Accounts checked; tomorrow prepared. |
| Analytics | Weekly | Metrics and URLs | Baseline table | Numbers captured; next experiment chosen. |
| Review | Weekly | Completed issues/cards | Decision log | Keep/change/stop decisions recorded. |
